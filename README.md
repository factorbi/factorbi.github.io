# Vansync Documentation

> Extract, transform and load data from legacy on-premises databases to AWS Aurora MySQL — unlocking modern BI tools like Amazon QuickSight and Google Data Studio for databases that lack native cloud connectivity.

## What is Vansync?

Vansync is a Windows ETL engine that bridges legacy on-premises databases — which have no native connectors to modern BI tools — to AWS Aurora MySQL, where the entire analytics ecosystem becomes available.

```
On-Premises DBs  →  Vansync Engine  →  AWS Aurora MySQL  →  BI Tools
(Firebird, dBase,                                           (QuickSight,
 VFP, Sybase,                                               Data Studio,
 SQL Server,                                                Redshift, ...)
 MySQL)
```

### Supported Source Databases

| Database | Notes |
|---|---|
| Firebird SQL | Used by Microsip, Aspel ERPs |
| Sybase SQL Anywhere | Used by Volante and others |
| dBase / DBF | Used by Maxirest and legacy POS systems |
| Visual FoxPro | |
| Microsoft SQL Server | Used by Intelisis ERP, Soft Restaurant |
| MySQL | Source-to-source sync or consolidation |

### Key Capabilities

- **Legacy Database ETL** — extract from databases with no native BI connectors
- **Multi-Location Consolidation** — merge data from multiple branches and DB engines into a single Aurora-MySQL instance
- **Scheduled Refresh** — configure sync schedules or trigger on-demand via API
- **Modern Data Warehouse** — load into Aurora-MySQL, Redshift, or any cloud-native DWH
- **Google Sheets Library** — pull spreadsheet data into your DWH via Google Apps Script
- **Two-way Synchronization** — optionally push data back from Aurora to on-premises tables
